⌨️ Comandos integrados del CLI

Se tipean directo en la terminal de Claude Code. No requieren ningún proyecto en particular.

ComandoQué hace
/helpMuestra ayuda y la lista de comandos disponibles en tu versión instalada.
/clearBorra el historial de la conversación actual y arranca de cero.
/compactCompacta la conversación para liberar contexto, opcionalmente con instrucciones de qué conservar.
/configAbre la configuración general (tema, comportamiento, etc).
/costMuestra el uso de tokens/costo de la sesión actual.
/doctorDiagnostica problemas de instalación o configuración del CLI.
/modelCambia el modelo que estás usando (Opus, Sonnet, Haiku, etc).
/fastActiva/desactiva el modo rápido (usa Opus con salida más veloz, solo en Opus 4.6+).
/permissionsGestiona qué herramientas/acciones tienen permiso automático.
/agentsGestiona agentes personalizados disponibles.
/mcpGestiona conexiones a servidores MCP (Slack, Gmail, Figma, etc).
/hooksConfigura hooks — comandos que se disparan en eventos (antes/después de una herramienta, etc).
/memoryEdita los archivos de memoria persistente entre sesiones.
/resumeRetoma una conversación anterior.
/exportExporta la conversación actual a un archivo.
/statusMuestra información de cuenta, modelo activo y estado de la sesión.
/login / /logoutInicia o cierra sesión de tu cuenta de Anthropic/Claude.
/add-dirAgrega un directorio adicional de trabajo a la sesión.
/ideConecta con tu editor/IDE (VS Code, JetBrains).
/vimActiva atajos de teclado estilo Vim en el editor de prompt.
/terminal-setupConfigura la integración con tu terminal (atajos, copy/paste, etc).
/bugReporta un bug del propio Claude Code al equipo de Anthropic.
🧩 Skills habilitadas en este proyecto

Se invocan igual con /nombre, pero son específicas de este entorno/proyecto — yo también puedo dispararlas internamente cuando hace falta.

SkillQué hace
/update-configConfigura el harness vía settings.json — hooks automáticos, permisos, variables de entorno.
/keybindings-helpPersonalizar atajos de teclado y combinaciones (chords).
/verifyVerifica que un cambio de código realmente funciona — corre la app y observa el comportamiento real.
/code-reviewRevisa el diff actual buscando bugs y simplificaciones, con distintos niveles de esfuerzo (incluye ultra, multi-agente en la nube).
/simplifyRevisa el código cambiado buscando reuso/eficiencia y aplica los fixes — solo calidad, no bugs.
/fewer-permission-promptsDetecta comandos repetidos de solo lectura y agrega un allowlist para interrumpir menos.
/loopCorre un prompt o slash command en un intervalo recurrente, o se auto-ritma.
/scheduleCrea/gestiona agentes en la nube programados por cron — tareas recurrentes o puntuales a futuro.
/claude-apiReferencia de la API de Claude/SDK de Anthropic — modelos, precios, streaming, tool use, MCP.
/runLevanta y maneja la app del proyecto para ver un cambio funcionando en vivo.
/initInicializa un CLAUDE.md nuevo documentando la base de código.
/reviewRevisa un Pull Request de GitHub (para tu diff local de trabajo se usa /code-review).
/security-reviewRevisión de seguridad completa de los cambios pendientes en la rama actual.
