map3d (MIT) — Modelo 3D exportable a formato GLB

Evaluación de si este repo open source puede reemplazar las APIs de Google Maps Platform que usa TMC para navegar publicaciones.

Investigado 23 jun 2026 · Fuente: github.com/cartesiancs/map3d (vía link compartido por Alejandro)
📌 Conclusión corta: NO reemplaza ninguna de las 3 funciones que usamos hoy. Queda documentado como referencia para el futuro, no para implementar ahora.

🗺️ Qué usa TMC hoy — Google Maps Platform

  • Maps JavaScript API — el mapa interactivo con los pines de cada publicación (frontend/components/mapa/MapaEspacios.tsx), pan/zoom, click en pin → ficha del espacio.
  • Places API (Autocomplete) — autocompletado de direcciones al publicar un espacio o editar el perfil (publicar/page.tsx, panel/page.tsx).
  • Geocoding (vía los datos que devuelve el propio Autocomplete) — convierte la dirección elegida en lat/lng para guardar el espacio y ubicarlo en el mapa.

📦 Qué es map3d

Aplicación web open source (MIT) que genera modelos 3D exportables (formato GLB) de edificios y calles de una ciudad real, usando datos de OpenStreetMap. Pensada para visualización 3D de una ciudad completa (tipo maqueta digital), no para navegación interactiva de marcadores.

  • Stack: React-Three-Fiber (Three.js + React), TypeScript 97.9%, Vite.
  • Distribución: open source, autohospedable. Demo público en map.fleet.im.
  • Madurez: 1.9k estrellas / 300 forks, pero solo 42 commits totales y documentación mínima (README básico). Roadmap con tareas pendientes (texturas de edificios, mapas de altura) — fase temprana.
  • Lo que NO tiene: autocompletado de direcciones, geocoding, pines clickeables con eventos, paneo/zoom de navegación tipo mapa de consumo.

🔮 Si en el futuro se quisiera dejar de depender de Google Maps Platform

map3d no es el camino, pero sí existe un ecosistema open source maduro que cubriría las 3 funciones, por si algún día se evalúa por costo:

  • Leaflet + tiles de OpenStreetMap — reemplazo directo de Maps JavaScript API (mapa interactivo, pines, pan/zoom). Gratis, muy usado en producción.
  • Nominatim o Photon — reemplazo de Places Autocomplete + Geocoding (búsqueda de direcciones, dirección → lat/lng). Gratis (Nominatim tiene límites de uso si se usa el servicio público; Photon es self-hostable).

Es un cambio de arquitectura más grande que migrar un repo puntual — implica tocar MapaEspacios.tsx, el formulario de publicar, y el preview de mapa en el panel. No se evaluó en profundidad en esta sesión; queda anotado para evaluar si Alejandro lo pide explícitamente.
