Protección del Código y la Marca
Documento interno · Mayo 2026

Este documento resume las medidas técnicas y legales recomendadas para proteger el software, la marca y la idea de negocio de TodasMisCosas.com antes y después del lanzamiento al público general.

1. Protección del Código en el Servidor

La arquitectura de TMC ya ofrece una protección natural muy significativa. La tabla siguiente resume qué partes del sistema están expuestas y cuáles no:

Capa del sistema ¿Está expuesta al público? Motivo
Backend (API Node.js) NO — Protegido ✅ Corre exclusivamente en el VPS de Hostinger. Ningún usuario externo puede descargarlo ni verlo.
Base de datos (MySQL) NO — Protegida ✅ No tiene acceso público. Solo accesible desde el propio servidor.
Repositorio en GitHub NO — Protegido ✅ Repositorio privado. Solo los colaboradores autorizados pueden verlo.
Frontend (JavaScript compilado) PARCIAL ⚠️ El navegador descarga el JS compilado. Sin embargo, está minificado y ofuscado — prácticamente ilegible. No se accede al código fuente original.
Conclusión técnica: La mayor parte del código está protegido por diseño. El frontend compilado es visible pero incomprensible en la práctica. No existe acción técnica urgente adicional requerida.

Buenas prácticas ya implementadas o a verificar

  • ✓ Repositorio de GitHub configurado como privado.
  • ✓ Archivo .env (claves secretas) nunca subido al repositorio.
  • ✓ Acceso SSH al VPS protegido con clave privada.
  • ! Verificar que solo Ale y Guille tengan acceso al repo y al servidor.
  • ! No compartir las credenciales de Hostinger, MySQL ni MercadoPago.

2. Protección de la Marca y el Nombre

El nombre "TodasMisCosas", el dominio y la identidad visual son los activos más valiosos y los más fáciles de copiar si no se protegen legalmente.

A) Registro de Marca en el INPI

El INPI (Instituto Nacional de la Propiedad Industrial) es el organismo oficial de Argentina para registrar marcas comerciales. La fecha de presentación de la solicitud te protege desde el día 1, aunque el trámite tarde 12–18 meses.
  • 1
    Ingresar a inpi.gob.ar y crear una cuenta como persona física o jurídica.
  • 2
    Buscar en la base de marcas si "TodasMisCosas" ya fue registrada por alguien más.
  • 3
    Presentar la solicitud en la clase 39 (servicios de almacenamiento y transporte) y posiblemente también en clase 42 (servicios tecnológicos / plataforma digital).
  • 4
    Costo aproximado: $50–$100 USD por clase. Se puede pagar online.

B) Copyright sobre el Código y Diseño

En Argentina, el derecho de autor sobre software existe automáticamente desde el momento de su creación (Ley 11.723). Para hacerlo oponible en juicio con mayor facilidad, se puede registrar formalmente:

DNDA — Dirección Nacional del Derecho de Autor
Registro de obra inédita (software + diseño visual).
Costo aproximado: muy bajo (~$20 USD). Web: argentina.gob.ar/justicia/dnda

C) Dominios Web

Asegurarse de tener registradas las variantes del dominio para que nadie las tome:

DominioEstado recomendado

3. Lo que NO se Puede Proteger

La idea de una plataforma de alquiler de espacios de almacenamiento entre particulares no es protegible como tal — existe en otros países bajo distintas formas. Lo que sí se protege es el nombre, el diseño, el código y la ejecución concreta del negocio.

Un competidor puede lanzar un negocio similar sin infringir ninguna ley. Lo que no puede hacer legalmente es:

  • ✗ Usar el nombre "TodasMisCosas" (si está registrado como marca).
  • ✗ Copiar el código fuente (protegido por copyright).
  • ✗ Reproducir el diseño visual / identidad de la plataforma.
  • ✗ Registrar todasmiscosas.com.ar si ya fue tomado por ustedes.

5. Resumen Ejecutivo

TMC ya está bien protegida técnicamente gracias a la arquitectura servidor-cliente y al repositorio privado.

La acción más importante antes del lanzamiento es presentar la solicitud de registro de marca ante el INPI para que el nombre "TodasMisCosas" quede reservado legalmente desde la fecha de presentación.

La ventaja competitiva real de TMC no está solo en el código: está en la red de usuarios, la confianza de la comunidad y la ejecución. Eso no se puede copiar.
